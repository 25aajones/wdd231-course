## Requirements

This project will be completed in a team of 2 students. After forming your team you will be responsible to plan and complete a web application as a showcase that demonstrates your mastery of the topics taught during this course. Your web application can take many different forms, but there is a list of requirements that each app must include:

- At least one detailed, validated form
- Local storage for persistence
- Fetching data from an API or json file.
- Drop-down menu or Modal
- CSS Animation
- Responsive design
- Good user experience (UX) and accessibility
- URL parameters (data that is passed from page to page in an app through the URL, for example https://example.com/page2.html?id=34 Send the `id` of `34` to the code running at `page2.html`)
- Use of modules for organization

## Checkpoint

This is the end. Turn in your completed project! Prepare a 3-5 minute video demo of your project to share with the instructor. In your video, be sure to cover:
- The problem your app solves and the key features you implemented.
- A demo of the key features in action.
- A brief reflection on the development process, including any challenges you faced and how you overcame them
- If you used AI: how you used it, what you accepted/rejected, and how you validated it. Did you generate less than 20% of your code with AI? If not, be prepared to explain why you chose to use more AI than recommended and how you validated the AI-generated code.
- Pick one feature that you implemented and that you are particularly proud of and explain why you like it. Explain also how you implemented it and any challenges you faced in doing so. Make sure to show and explain the code for this feature in your video. HTML, CSS and Javascript.
