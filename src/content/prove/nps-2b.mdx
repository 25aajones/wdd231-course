## Scope and constraints

- Work individually.
- Use the provided project structure and static dataset.
- Keep the page focused on the three content areas:
  - Current Conditions
  - Fees and Passes
  - Visitor Centers
- Build for maintainability: this is part of a multi-step project.

## Deliverables

1. A completed homepage (`index.html` + JS + CSS) matching the target layout and behavior.
2. Dynamic rendering of park-specific content from data (not hardcoded content text in main sections).
3. Refactored, readable implementation that is easy to extend in future assignments.

## Acceptance criteria

Your work is complete when all are true:

### Data-driven content

- Park name and summary content are rendered from data.
- Three info cards are rendered from a data structure (not manually duplicated HTML blocks).
- Footer contact details render from the correct records in data:
  - Mailing address
  - Voice phone number

### UX and layout

- Layout matches the provided mockups at narrow and wide widths.
- Content hierarchy is clear and readable.
- Images behave responsively.

### Code quality

- Major rendering concerns are separated into focused functions/modules.
- Repetition is minimized (templates/helpers where appropriate).
- Naming is clear and consistent.

### Verification

- `npm run start` works as expected.
- `npm run build` and `npm run preview` run successfully and match expected behavior.

## Suggested approach (professional workflow)

Use this workflow as guidance, not a strict checklist:

1. **Read the data model first**
   - Identify the exact properties needed for intro, cards, and footer.
   - Decide how to map data to UI components.

2. **Define rendering contracts**
   - Choose which functions are responsible for:
     - header-related updates,
     - intro rendering,
     - card rendering,
     - footer rendering.

3. **Build vertically in small slices**
   - Complete one visible section end-to-end before moving to the next.
   - Verify each slice in the browser before continuing.

4. **Refactor after functionality works**
   - Extract reusable templates/helpers.
   - Keep module boundaries clean for future pages.

5. **Run production checks before submission**
   - Confirm build and preview output.
